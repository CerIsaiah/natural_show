What this project is

This is my take on how OpenClaw and Natural fit together. OpenClaw already runs a lot of real assistant workflows — but moving money is still a gap for most setups. Natural is the infrastructure that closes it, with delegation, limits, and compliance built in. The flow here is: natural language → tool calls → balance checks → payment, with a ledger and audit trail so you can see the whole story.

It has two layers: an installable OpenClaw skill (real SKILL.md, real tool handlers against Natural's API) and a live web demo so anyone can try the agent flow without a key.

01

OpenClaw Payment Skill

An installable skill that connects any OpenClaw assistant to Natural's payment API. 5 tools, proper SKILL.md format, zero dependencies. Set a key and go.

02

Live Agent Demo

An interactive agent you can chat with — real-time tool calls, a live ledger, and a full audit trail. Shows the payment flow from natural language to execution.

03

Payment Orchestration Engine

A custom-built ledger with delegation enforcement, per-transaction and daily spending limits, idempotency, and duplicate detection. Built from scratch.

04

Agent Tool Interface

Natural language parsing that extracts amounts, recipients, and memos from messy input. The agent decides which tools to call and in what order.

How the agent flow works

This is what I think the payment UX should look like when agents are the interface:

User:  pay terri 500 for fixing the bathroom sink

Agent: → Checking balance for Bruce Property Management...
       → Balance: $12,500.00 available
       → Sending $500 to terri@plumbingworks.com...
       → Payment created: pay_a8f3k2m1 (PENDING)

       ✓ Sent $500.00 to Terri
         Memo: fixing the bathroom sink
         Status: PENDING

User:  pay terri 500 for the bathroom again

Agent: → ⚠️ Duplicate detected: similar payment to Terri in the last hour
       → Proceeding with warning...

User:  pay jake 10000 for the full rewire

Agent: → ❌ Blocked: $10,000 exceeds per-transaction limit ($5,000)
       → Payment not sent. Ask the business to raise the limit.

The agent checks balance before sending, enforces spending limits, flags likely duplicates, and returns clear status. Everything is logged — the kind of observability you'd want for real money movement.

You can use this in OpenClaw today

If you run OpenClaw, add the skill and set your Natural API key — it talks to api.natural.co with the right JSON:API shape and idempotency headers:

$ openclaw skills add https://github.com/CerIsaiah/natural_show

Installing natural-payments...
Found SKILL.md with 5 tools
✓ Installed.

$ /secrets set NATURAL_API_KEY sk_ntl_sandbox_your_key

The handlers use Node's built-in fetch — no extra dependencies. With a valid key, you're on the real API.
